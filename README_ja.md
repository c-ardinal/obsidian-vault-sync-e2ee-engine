# Vault-Sync E2EE Engine

[ [🇺🇸 English](README.md) | [🇯🇵 日本語](README_ja.md) ]

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

[Vault-Sync](https://github.com/c-ardinal/obsidian-vault-sync) 専用の外部E2EEエンジンです。  
Vault-Sync がアップロードする全ての Vault データを暗号化し、ダウンロード時にローカル上で復号します。  
クラウド側で平文を見る術はありません。

---

## ✨ 機能

- **AES-256-GCM 暗号化**:
    - 業界標準の認証付き暗号化ですべての Vault ファイルを保護
- **PBKDF2 鍵導出**:
    - パスワードから 600,000 回のイテレーションでマスターキーを導出
- **ファイルごとのランダム IV**:
    - 暗号化ごとに一意の初期化ベクトルを使用し、暗号文のパターン分析を防止
- **パスワード強度チェッカー**:
    - セットアップ時に強度インジケータを表示（zxcvbn ベースのスコアリング）
- **自動ロック解除**:
    - Obsidianのセキュアストレージ（Keychain）にパスワードを保存し、起動時の入力を省略可能
- **透過的な統合**:
    - Vault-Sync のスマート同期（3-way マージ、競合検出、履歴表示）が暗号化データでもシームレスに動作
- **ワンクリック移行**:
    - セットアップウィザードが既存 Vault データの暗号化と整合性検証を自動で処理

## ⌨️ コマンド（コマンドパレット経由）

E2EE Engine をインストール後、Obsidian のコマンドパレットから以下のコマンドが利用可能になります:

| コマンド                                | 説明                                         |
| --------------------------------------- | -------------------------------------------- |
| E2EE: Vaultの暗号化を開始する           | 非暗号化 Vault で E2EE を初期化              |
| E2EE: Vaultのロックを解除する           | ロック中の Vault をパスワードで解除          |
| E2EE: 暗号化パスワードを変更する        | マスターキーを新パスワードで再ラッピング     |
| E2EE: リカバリーコードを表示する        | リカバリーコードと鍵フィンガープリントを表示 |
| E2EE: リカバリーコードでVaultを復元する | リカバリーコードで Vault アクセスを復元      |

## 🚀 インストール

注意: E2EEを有効化した場合、無効化することはできません。E2EEを使用しない状態で本プラグインを使用し続けたい場合は [#復旧手順](#️復旧手順) を実行して下さい。

### リリースから

1. [リリースページ](https://github.com/c-ardinal/obsidian-vault-sync-e2ee-engine/releases) から最新版の `e2ee-engine.js` をダウンロード
    - 最新版以外のバージョンは使用できません。インストールしてもVault-Syncは本ファイルをロードしません。
2. Vault-Sync のプラグインディレクトリに配置:
    ```
    <Your Vault>/.obsidian/plugins/obsidian-vault-sync/e2ee-engine.js
    ```
3. Obsidian を再起動
4. コマンドパレットから暗号化開始コマンドを実行
5. E2EE セットアップウィザードが表示されます。指示に従ってパスワード設定と Vault 移行を進めて下さい

### ソースから

1. 依存関係をインストール:

    ```bash
    npm install
    ```

2. エンジンをビルド:

    ```bash
    npm run build
    ```

3. `dist/e2ee-engine.js` をプラグインディレクトリにコピー:
    ```
    <Your Vault>/.obsidian/plugins/obsidian-vault-sync/e2ee-engine.js
    ```

## 🔒 セキュリティアーキテクチャ

### Vault Lock

`vault-lock.vault` ファイルは暗号化された Vault と共に保存され、以下を含みます:

- 暗号化されたマスターキーマテリアル
- PBKDF2 のソルトとイテレーション数
- パスワードヒント（設定した場合）

このファイルは Vault のロック解除に必要です。このファイルとパスワードがなければ、データを復号することはできません。  
このファイルはローカルには保管されず、クラウドストレージ上の以下の場所に格納されます。

```
  <Your Vault>/.obsidian/plugins/obsidian-vault-sync/data/remote/vault-lock.vault
```

## 🔧 技術仕様

| 項目         | 値                                                          |
| ------------ | ----------------------------------------------------------- |
| フォーマット | CommonJS (CJS)                                              |
| 暗号化方式   | AES-256-GCM（Web Crypto API）                               |
| 鍵導出       | PBKDF2-SHA256、600,000 イテレーション                       |
| IV サイズ    | 12 バイト（操作ごとにランダム生成）                         |
| ローダー     | Vault-Sync がセキュアな `new Function` 評価器で動的にロード |

## ⚠️ 重要な注意事項

- **パスワードを絶対に忘れないでください**:
    - パスワードとリカバリーコードの両方を失った場合、暗号化されたデータを復号することはできません。「リカバリーコードを表示する」コマンドでバックアップ用リカバリーコードをエクスポートし、安全な場所に保管してください。
    - 開発者であっても復元は不可能です。
- **vault-lock.vault をバックアップしてください**:
    - クラウドストレージにのみ存在します。このファイルは復号に不可欠です。紛失するとデータを復元できません。
    - 開発者であっても復元は不可能です。
- **マルチデバイス**:
    - Vault を共有するすべてのデバイスで同じパスワードを使用する必要があります。あるデバイスで E2EE を有効にすると、他のデバイスでは次回同期時にパスワードの入力を求められます。

## ❗️トラブルシューティング

### パスワードを忘れた/リカバリーコードを紛失した/vault-lock.vaultを紛失した場合

クラウドストレージ上のデータは**如何なる手段でも復号化できません**。そのため、同期や履歴参照などの機能も利用できません。  
E2EEを使用しない状態で本プラグインを使用し続けたい場合は [#復旧手順](#️復旧手順) を実行して下さい。

### E2EEを無効化したい場合

一度有効化したE2EEを無効化することはできません。  
E2EEを使用しない状態で本プラグインを使用し続けたい場合は [#復旧手順](#️復旧手順) を実行して下さい。

### 復旧手順

注意: 以下の手順を実行すると、プラグインの設定やファイルの変更履歴など、**Vault-Syncプラグイン関連のデータは全て失われます**。

1. Obsidian外の操作:ブラウザやクラウドストレージのアプリから、クラウドストレージ上のVaultフォルダ(デフォルト設定の場合、ObsidianVaultSync配下)を削除する。
2. Obsidianの設定 > コミュニティプラグイン > インストールされたプラグイン からVault-Syncをアンインストールする。
3. Obsidianを再起動する
4. Obsidianを起動し、設定 > コミュニティプラグイン > 閲覧 からVault-Syncを再インストールする。
5. Vault-Syncの設定からログインし、Vaultをクラウドストレージへアップロードする。
