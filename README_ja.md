# VaultSync E2EE Engine

[ [🇺🇸 English](README.md) | [🇯🇵 日本語](README_ja.md) ]

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

[VaultSync](https://github.com/c-ardinal/obsidian-vault-sync) 用の外部エンドツーエンド暗号化エンジンです。  
導入すると、すべての Vault データがアップロード前にデバイス上で暗号化され、ダウンロード後にローカルで復号されます。  
クラウド側で平文を見る術はありません。

---

## ✨ 機能

- **AES-256-GCM 暗号化**: 業界標準の認証付き暗号化ですべての Vault ファイルを保護
- **PBKDF2 鍵導出**: パスワードから 600,000 回のイテレーションでマスターキーを導出
- **ファイルごとのランダム IV**: 暗号化ごとに一意の初期化ベクトルを使用し、暗号文のパターン分析を防止
- **パスワード強度チェッカー**: セットアップ時に強度インジケータを表示（zxcvbn ベースのスコアリング）
- **自動ロック解除**: Obsidianのセキュアストレージ（Keychain）にパスワードを保存し、起動時の入力を省略可能
- **透過的な統合**: VaultSync のスマート同期（3-way マージ、競合検出、履歴表示）が暗号化データでもシームレスに動作
- **ワンクリック移行**: セットアップウィザードが既存 Vault データの暗号化と整合性検証を自動で処理

## 🚀 インストール

### リリースから

1. [リリースページ](https://github.com/c-ardinal/obsidian-vault-sync-e2ee-engine/releases) から `e2ee-engine.js` をダウンロード
2. VaultSync のプラグインディレクトリに配置:
    ```
    <Your Vault>/.obsidian/plugins/obsidian-vault-sync/e2ee-engine.js
    ```
3. Obsidian を再起動
4. コマンドパレットからE2EE設定コマンドを実行します
5. E2EE セットアップウィザードが表示されます。指示に従ってパスワード設定と Vault 移行を完了してください

### ソースから

1. 依存関係をインストール:

    ```bash
    npm install
    ```

2. エンジンをビルド:

    ```bash
    npm run build
    ```

3. `dist/e2ee-engine.js` をプラグインディレクトリにコピー:
    ```
    <Your Vault>/.obsidian/plugins/obsidian-vault-sync/e2ee-engine.js
    ```

## 🔒 セキュリティアーキテクチャ

### Vault Lock

`vault-lock.vault` ファイルは暗号化された Vault と共に保存され、以下を含みます:

- 暗号化されたマスターキーマテリアル
- PBKDF2 のソルトとイテレーション数
- パスワードヒント（設定した場合）

このファイルは Vault のロック解除に必要です。このファイルとパスワードがなければ、データを復号することはできません。  
このファイルはローカルには保管されず、クラウドストレージ上の以下の場所に格納されます。

```
  <Your Vault>/.obsidian/plugins/obsidian-vault-sync/data/remote/vault-lock.vault
```

## 🔧 技術仕様

| 項目         | 値                                                         |
| ------------ | ---------------------------------------------------------- |
| フォーマット | CommonJS (CJS)                                             |
| 暗号化方式   | AES-256-GCM（Web Crypto API）                              |
| 鍵導出       | PBKDF2-SHA256、600,000 イテレーション                      |
| IV サイズ    | 12 バイト（操作ごとにランダム生成）                        |
| ローダー     | VaultSync がセキュアな `new Function` 評価器で動的にロード |

## ⚠️ 重要な注意事項

- **パスワードを絶対に忘れないでください**: パスワード復旧の仕組みはありません。パスワードを忘れた場合、暗号化されたデータを復号することはできません。
- **vault-lock.vault をバックアップしてください**: このファイルは復号に不可欠です。紛失するとデータを復元できません。
- **マルチデバイス**: Vault を共有するすべてのデバイスで同じパスワードを使用する必要があります。あるデバイスで E2EE を有効にすると、他のデバイスでは次回同期時にパスワードの入力を求められます。
